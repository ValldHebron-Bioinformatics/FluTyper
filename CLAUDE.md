# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

FluTyper is a Nextflow DSL2 pipeline for genotyping zoonotic influenza viruses (avian and swine) and characterizing relevant mutations, designed for genomic surveillance within a One Health framework.

## Running the Pipeline

```bash
# Run with default test data
nextflow run nf_pipeline/main.nf

# Run with custom input
nextflow run nf_pipeline/main.nf --inputFasta <your_fasta.fasta> --protocol AVIAN --outDir <output_dir>

# Resume a failed run
nextflow run nf_pipeline/main.nf -resume
```

Default parameters (from `nf_pipeline/nextflow.config`):
- `--inputFasta`: `docs/fastas/prova.fasta`
- `--protocol`: `AVIAN`
- `--outDir`: `prova/1/`

There is no formal test suite. The test data is `docs/fastas/prova.fasta`.

## Input/Output Format

**Input**: MultiFASTA with headers using `_` or `|` as separator:
- `>{SequenceID}_{Segment}_{OptionalInfo}` (e.g., `>19135815_HA_2022_Spain`)
- Supported segments: HA, NA, PB2, PB1, PA, NP, MP, NS

**Output** (in `--outDir`):
- `final_genotyping_results.csv` — SampleID, Subtype, Dataset, Version, Clade, QC status
- `final_mutations_report.xlsx` — All mutations, one sheet per protein
- `relevant_mutations.xlsx` — Filtered: mutations in ≥10% of samples OR flagged as MARKER=TRUE
- `pipeline_errors.log` — Aggregated errors from all modules
- `samples/{sample_id}/` — Per-sample intermediate files (segments, CDS, proteins, mutations)

## Architecture

The pipeline has 10 sequential Nextflow processes in `nf_pipeline/modules/`:

```
OrganizeBySample → SubtypeDetection → GetDatasets → GenotypingNextclade → GenotypingResults
                                                                                ↓
                 MutationsCompiler ← MutationsFinder ← TranslateToProtein ← GetCDS
                        ↓
               CompileErrors (collects error logs from all stages)
```

**Key design points:**

- All processes use `errorStrategy 'ignore'` — the pipeline continues even when individual samples fail. Errors are written to per-module log files (e.g., `SDerrors.log`, `CDSerrors.log`) and aggregated by `CompileErrors` into `pipeline_errors.log`.

- **SubtypeDetection** uses a minimizer index (`protocols/AVIAN/v1/AVIAN_minimizers.json`) for fast H/N subtype assignment, then infers pathotype (HPAI/LPAI) for H5/H7/H9.

- **GetCDS** uses MAFFT alignment against reference CDS sequences, then trims gaps. NEP and M2 proteins have strict gap thresholds (400, 600 bp) to prevent frameshifts.

- **MutationsFinder** compares translated proteins to H/N-specific references, annotates known markers from `protocols/AVIAN/v1/MARKERS/*.csv`, and can translate HA positions across H subtypes using `bin/MutationsDictionary.py`.

- **MutationsCompiler** (Python/pandas/openpyxl) produces the two Excel reports. The "relevant" report filters by: `MARKER == TRUE` OR mutation frequency ≥ 10% across all samples.

- Only **H5** is fully implemented end-to-end. H7 and H9 produce empty genotyping result files as placeholders. SWINE protocol is not yet implemented.

## Protocol Resources

`protocols/AVIAN/v1/` contains all reference data:
- `AVIAN_minimizers.json` — k-mer minimizer index for subtyping (generated by `bin/create_minimizer_index.py`)
- `Avian_references.fasta`, `CDS_references.fasta`, `PROT_references.fasta` — reference sequences
- `AA_Sites.csv` — amino acid numbering correspondence across H subtypes
- `MARKERS/*.csv` — known mutations per protein (columns: position, AA, origin, effect, marker flag)

## Python Utilities

**`bin/MutationsDictionary.py`**: Translates amino acid positions between H subtypes using `AA_Sites.csv`. Called from within `MutationsFinder.nf` with `--subtype`, `--input`, `--dictionary`, `--base`, `--output` arguments.

**`bin/create_minimizer_index.py`**: Builds the minimizer JSON index from reference FASTAs. Requires a YAML config (`protocols/AVIAN/v1/config.yaml`). Run this when updating reference sequences.

## Dependencies

- Nextflow 2.0+ (DSL2)
- seqkit, MAFFT, Nextclade
- Python 3: `pandas`, `biopython`, `openpyxl`, `click`, `pyyaml`, `numpy`
